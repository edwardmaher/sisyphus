# Tracking

## Features
- Daily usage detection on boot
- Missed-day yelling mode activation
- Apology collection and genuineness evaluation (via Gemma)
- Auto-accept after 5 apologies
- Animation state coordination

## Usage Detection
On every boot:
1. Check `~/.sisyphus/data/` for a record dated yesterday (`YYYY-MM-DD.json`).
2. If absent → missed day detected → trigger yelling mode.
3. If present → normal mode.

Edge: first ever run → no history → normal mode (no yelling on day 1).

## Yelling Mode
- Switch Sisyphus animation to "yelling" state.
- Before any daily review messages, Sisyphus delivers a tirade mixing English and ancient Greek curses. Text generated by Gemma using prompt in `src/prompts/yelling-tirade.txt`.
- User must submit an apology.
- Gemma evaluates genuineness: prompt in `src/prompts/apology-eval.txt`, returns `{"genuine": true|false, "response": "..."}`.
- If genuine: Sisyphus accepts, switches back to pushing animation, proceeds to daily review.
- If not genuine: Sisyphus responds with escalated disappointment (Gemma-generated), re-prompts.
- After 5 total apology attempts (regardless of Gemma result): auto-accept, proceed.

## Apology Tracking
Store attempt count in memory (not persisted). Reset on app exit.

## State Machine
```
BOOT
  → check yesterday record
  → [missing] → YELLING_MODE → apology loop → DAILY_REVIEW
  → [present]  → DAILY_REVIEW
```

## Output / Side Effects
- Sets global UI animation state: "pushing" | "yelling"
- Returns `{ missedDay: boolean, apologyAccepted: boolean }`
