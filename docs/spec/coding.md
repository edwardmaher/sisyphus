# Coding

## Features
- Git commit discovery for previous day
- Project completion grade (1/5 threshold)
- Code quality value grade via Gemma
- Trend commentary vs. yesterday, this week, this month

## Git Review
Use `git log` across all repos found in a configurable list (`~/.sisyphus/config.json` → `"repos": ["/path/to/repo"]`).
On first run if no repos configured, prompt user to add repo paths.

Command:
```bash
git -C {repoPath} log --since="yesterday 00:00" --until="today 00:00" --pretty=format:"%H %s" --author="$(git config user.email)"
```
Collect all commit SHAs. For each commit fetch diff:
```bash
git -C {repoPath} show {sha} --stat --unified=3
```
Concatenate diffs (truncate at 8000 chars to stay within Gemma context).

## Completion Grade
Count total commits across all repos for the day.
Threshold: ≥ 1 commit that changes ≥ 50 lines net = "adequate" (represents roughly 1/5 project progress per day heuristic).
If total changed lines < 50 and commit count < 3 → "inadequate".
Edge: no commits → "inadequate", note "No commits found yesterday."

## Value Grade (via Gemma)
Send diff summary to Gemma with prompt from `src/prompts/coding-review.txt`.
Gemma returns: letter grade [F,C-,C,C+,B-,B,B+,A-,A] + 1–2 sentence feedback.
Parse response: expect `Grade: X\nFeedback: ...` format.
Fallback if parse fails: show raw response, grade = null.

## Trend Commentary
Load previous day's record, last 7 days average, last 30 days average from `~/.sisyphus/data/`.
If data absent for a period: skip that period's comparison silently.
Compare value grade numerically (F=0, C-=1, C=2, ... A=8).
Commentary generated by Gemma in same call or post-processed locally.

## Output Shape
```ts
{
  completionGrade: "adequate" | "inadequate",
  valueGrade: "A" | "B+" | ... | null,
  feedback: string,
  commitCount: number,
  linesChanged: number
}
```
